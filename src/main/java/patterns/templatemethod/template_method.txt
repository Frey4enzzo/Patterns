Шаблонный метод определяет основные шаги алгоритма и позволяет
субклассам предоставить реализацию одного или нескольких шагов.

Паттерн Шаблонный Метод задает "скелет" алгоритма в методе,
оставляя определение реализации некоторых шагов субклассам.
Субклассы могут переопределять некоторые части алгоритма без изменения
его структуры.

Основной задачей паттерна является создание шаблона алгоритма.

Объявляется ключевым слово final, чтобы субклассы не могли изменить
последовательность шагов в алгоритме.

Arrays.sort() - шаблонный метод, объект должен реализовать интерфейс Comparable

"Перехватчик" - метод абстрактного класса с пустой реализацией. При
помощи перехватчика субкласс может реализовать необязательную часть алгоритма.

"Голливудский прицип" - не вызывайте нас, мы сами вас вызовем. Помогает
предотвратить "разложение зависимостей".

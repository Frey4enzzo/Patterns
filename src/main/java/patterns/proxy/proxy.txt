Заглушка - клиентский помощник (заместитель)
Скелет - серверный помощник

Действия для создания удаленной службы (RMI):
1. Создание интерфейса удаленного доступа (определяет методы, вызываемые клиентом в удаленном режиме)
2. Создание реализации интерфейса удаленного доступа (выполняет настоящую работу)
3. Генерирование заглушек и скелетов утилитой rmic
4. Запуск rmiregistry (клиент обращается к реестру RMI для получения заместителя/заглушки/объекта-помощника)
5. Запуск удаленной службы
